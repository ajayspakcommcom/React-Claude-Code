### ⚠️ INTERMEDIATE — Routing (PARTIAL — resume here next session)

**Decision**: Used TanStack Router (more powerful, 100% type-safe) over React Router.
**Two modes explored**: Code-based + File-based.
**Routing lives in its own project**: `React-App-FileRouter/` (Vite-based)

#### Why separate project?
- `@tanstack/router-plugin` watches `src/routes/` and auto-generates `routeTree.gen.ts`
- Works best with Vite (Webpack support is experimental)
- Keeps `React-App` clean

#### Run file-based routing app:
```bash
cd React-App-FileRouter
npm run dev        # → http://localhost:5173
```

#### Concepts covered in React-App-FileRouter:
| Concept | How |
|---------|-----|
| File → Route mapping | `about.tsx` = `/about`, `$userId.tsx` = `/:userId` |
| Root layout | `__root.tsx` — nav + `<Outlet />` |
| Index route | `index.tsx` inside a folder |
| Nested routes | `users/index.tsx` + `users/$userId.tsx` + `<Outlet />` |
| Dynamic params | `$userId.tsx` → `params.userId` (fully typed) |
| Route loader | `loader` in `createFileRoute` — fetch before render |
| Pathless layout | `_auth.tsx` — groups protected routes, no URL segment |
| Protected routes | `beforeLoad` guard → `redirect` to `/login` |
| Search params | `validateSearch: z.object(...)` — typed query params |
| Auto-generated tree | `routeTree.gen.ts` — never edit manually |
